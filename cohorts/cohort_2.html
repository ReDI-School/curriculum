Redi School curriculum cohort 2

Aims and goals

  • An ability to use a linux machine with fluency and confidence. This means:
    • Basic command line knowledge ls cd mkdir touch apt-get , what sudo does
    • User management, ability to install packages through the gui
    • Understanding that linux != ubuntu
  • Understanding how to program for the web in JS. This means:
    • expressions, variable assignments, arithmetic
    • Array comprehensions, inheritance, functions
    • debugging, working out how to do things, googling
    • debugging, working out how to do things, googling
  • The ability to create and deploy a website. This means:
    • Learning enough HTML and CSS to express a sense of design through code
    • Having a basic understanding of the box model, and HTML nesting
    • Understanding CSS, and maybe a preprocessor (SASS?)
    • Knowing enough git to deploy to github pages, and to solve merge conflicts

Introductory Lessons

These would be the lessons for students who haven't done coding before, these wouldn't necessarily be applicable to all students

Lesson 1 - welcome to your new Ubuntu machine!

The intent of this lesson is to make sure that all students are at the same level of computer literacy. It may not be appropriate for all groups, particularly those who have done some programming before

learning outcomes:

    Student will:
  • - understand that this is different to a windows machine, and that the OS is different, what an OS is
  • - understand how to install new programs on the machine, demonstration through installing a .deb using the gui, and using apt-get
  • - learn to navigate through the file system using the command line including cd,touch, mkdir, rm (-r)
  • ... anything extra needed?

Teachers should be have familiarity with a *nix machine

Lesson 2 - welcome to the internet

The intent of this lesson is to get the students to understand basically what the internet does, and have their hand in creating it!

learning outcomes:

    Student will:
  • - understand the server / client model of the internet (i.e. understand what is going on when we make a request)
  • - understand URLs, be encouraged to think critically, about what is likely going on when we use the internet (i.e. have some familiarity about web tech stacks?)
  • - be able to create simple information focussed websites using common tags e.g.h1 a p div
  • - understand what the CSS selectors . and #

Lesson 3 - supercharged styling

The intent of this lesson is to get the students to be able to express something individualistic and creative. they should start to be able to express almost any design intent using CSS. They should be able to google their problems, and be encouraged to spend their spare time making their website better

learning outcomes:

    Student will:
  • - start to understand the box model of CSS
  • - understand the rules of specificity of css selectors i.e. what div.my-class a means. By this point students should be able to select any arbitrary DOM node on the page
  • - understand how flex works, and how it makes layouting simpler
This lesson might be difficult for teachers/developers that don't work with CSS every day. Recommended reading: here, teachers should also prepare for this lesson by implementing the Holy Grail layout
